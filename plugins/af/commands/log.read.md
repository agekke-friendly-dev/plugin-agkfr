# セッションログ読み込み [vba,py,pyqt]

セッション開始時に必須ファイルと最新ログを読み込む。

## 引数
- $ARGUMENTS: `vba` | `py` | `pyqt`（言語ガイド、複数可）

## 手順

### 1. 情報収集スクリプト実行

```bash
python ${CLAUDE_PLUGIN_ROOT}/scripts/log_read.py $ARGUMENTS
```

出力内容:
- 読み込み対象ファイル一覧

### 2. ファイル読み込み

スクリプト出力の「読み込み対象ファイル」を**並列Read**で読み込む。

### 2a. プロジェクト必須ファイル読み込み（スクリプト出力とは別に必ず実行）

**スクリプト出力に含まれていなくても、以下のファイルが存在すれば必ず並列Readする:**

| # | ファイル | 重要度 | 読み込み後のアクション |
|---|----------|--------|----------------------|
| 1 | `./CLAUDE_IMPORTANT.md` | **最重要** | 内容を報告セクションに**全文引用**して表示。ここに書かれた制約・ルールはセッション中厳守すること |
| 2 | `./TODO.md` | 重要 | 未完了タスク数と直近の保留事項を報告に含める |

**CLAUDE_IMPORTANT.md について:**
- プロジェクトで特に重要な事項をまとめたファイル
- セッション開始時に必ず確認し、記載内容を把握してから作業を開始する
- 報告では「CLAUDE_IMPORTANT.md の内容」として独立セクションで表示する

### 3. 報告（最後にまとめて）

**ユーザーへの出力を停止する直前に、以下をまとめて報告:**

- セッション履歴（最新4件）
- **CLAUDE_IMPORTANT.md の内容**（存在する場合、重要事項を箇条書きで表示）
- **TODO.md 概要**（存在する場合、未完了タスク数・保留事項の要約）
- 前回セッションの作業概要・引継ぎ事項（ログから要約）
- 質問:「続きから作業しますか？」または「何から始めますか？」

### 4. cwd確認（最後に必ず実行）

`pwd` でカレントディレクトリを確認し、**作業対象のプロジェクトフォルダ**になっていることを検証する。

- 一致 → そのまま完了
- 不一致 → `cd` で正しいプロジェクトフォルダに移動してから完了
